# ORO CRM (2단계 — Supabase 연동)

ORO 주식회사의 거래처/영업 관리 CRM입니다. 기존 MES 앱과는 **완전히 독립된 프로젝트**로,
이 `crm/` 폴더 안에서 자체적으로 실행/빌드됩니다.

## 저장 모드 (두 가지)

| 모드 | 저장 위치 | 로그인 | 데이터 공유 |
|------|-----------|--------|-------------|
| ☁️ **클라우드** (기본) | Supabase 서버 (MES와 같은 프로젝트) | MES 계정 그대로 사용 | 팀원 모두 같은 데이터 |
| 💾 **로컬** | 브라우저 localStorage | 불필요 | 이 브라우저에서만 |

- 앱을 열면 로그인 화면이 나옵니다. **MES에서 쓰던 이메일/비밀번호** 그대로 로그인하면 됩니다.
- 로그인 없이 써보려면 로그인 화면의 "로컬 모드로 사용" 버튼을 누르세요 (예시 데이터로 시작).
- 사이드바 하단에서 언제든 모드를 전환할 수 있습니다.

## 지금 되는 것
- 거래처 / 담당자 / 딜(영업기회) / 대화 기록 등록 → **서버 저장, 어느 기기에서든 로그인하면 보임**
- 이메일·LINE·WeChat·전화·메모 채널별 대화 타임라인
- 영업 파이프라인 칸반 (문의 → 견적 → 샘플 → 평가 → 승인 → 양산)

## 아직 안 되는 것 (다음 단계)
- 메일 자동 수집 (서버 연동 필요)
- 다른 사람이 수정한 내용의 실시간 반영 (현재는 새로고침해야 보임)
- 수정/삭제 기능 (현재는 추가와 딜 단계 이동만 가능)

## 실행 방법
```bash
cd crm
npm install
npm run dev      # 개발 서버 (http://localhost:5173)
npm run build    # 배포용 빌드 (dist/ 생성)
```

## Supabase 구성
- 프로젝트: `oro-mes` (fzoombsxvscndzrhzmwb) — MES와 같은 프로젝트, 같은 로그인 계정
- 테이블: `crm_companies`, `crm_contacts`, `crm_deals`, `crm_activities` (마이그레이션 `create_crm_tables`로 적용됨)
- 보안: RLS 활성화 — **로그인한 사용자만** 읽고 쓸 수 있음 (MES와 동일한 정책)
- 스키마 전문은 `supabase/schema.sql` 참고 (다른 프로젝트에 새로 설치할 때 사용)
- 다른 Supabase 프로젝트를 쓰려면 `.env.example`을 `.env`로 복사해 값을 채우세요.
