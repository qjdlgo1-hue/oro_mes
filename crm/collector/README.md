# CRM 메일 자동 수집기 (네이버 메일 + 네이버 웍스)

GitHub Actions가 **1시간마다** 네이버 메일서버(IMAP)에 접속해, CRM에 등록된 거래처
도메인과 주고받은 메일을 찾아 CRM 타임라인에 자동 기록합니다.
Claude나 다른 서비스 없이 GitHub + Supabase만으로 독립적으로 돌아갑니다.

## 동작 방식
1. CRM 거래처 목록에서 "이메일 도메인"이 채워진 회사들을 가져옴
2. 네이버 메일/네이버 웍스의 받은편지함 + 보낸편지함에서 최근 3일치 메일 확인
3. 보낸/받는 사람 주소의 도메인이 거래처 도메인과 일치하면 타임라인에 기록
   - 거래처에서 온 메일 → "받음", 우리가 보낸 메일 → "보냄"
   - 같은 메일은 두 번 기록되지 않음 (메시지 ID 기반 중복 방지)

## 최초 설정 (한 번만)

### 1. 네이버 메일 — IMAP 켜기
1. https://mail.naver.com 접속 → 오른쪽 위 ⚙(환경설정) → **POP3/IMAP 설정** → **IMAP/SMTP 사용: 사용함** 저장
2. 네이버 2단계 인증을 쓰고 있다면: 네이버 내정보 → 보안설정 → **애플리케이션 비밀번호** 발급
   (이 경우 아래 Secrets의 비밀번호 자리에 애플리케이션 비밀번호를 넣습니다)

### 2. 네이버 웍스 — IMAP 켜기
1. 네이버 웍스 메일 → 환경설정 → **IMAP/SMTP(외부 메일 프로그램) 사용** 켜기
   (안 보이면 관리자 콘솔에서 허용이 필요할 수 있습니다)
2. 2단계 인증 사용 시 앱 비밀번호를 발급해 사용

### 3. GitHub Secrets 등록
GitHub 저장소 → **Settings → Secrets and variables → Actions → New repository secret** 에서 아래를 등록:

| Secret 이름 | 값 |
|---|---|
| `SUPABASE_SERVICE_ROLE_KEY` | Supabase 대시보드 → 프로젝트 oro-mes → Settings → API → `service_role` 키 |
| `NAVER_MAIL_USER` | 네이버 아이디 (예: `qjdlgo1` — @naver.com 앞부분) |
| `NAVER_MAIL_PASSWORD` | 네이버 비밀번호 (2단계 인증 시 애플리케이션 비밀번호) |
| `WORKS_MAIL_USER` | 네이버 웍스 이메일 전체 주소 (예: `dwlee@orocorp.kr`) |
| `WORKS_MAIL_PASSWORD` | 네이버 웍스 비밀번호 (또는 앱 비밀번호) |

- 두 계정 중 하나만 등록해도 됩니다 (등록된 계정만 수집).
- ⚠️ `service_role` 키는 DB 전체 권한이 있으니 **GitHub Secrets에만** 넣고 코드나 채팅에 붙여넣지 마세요.

### 4. 테스트
GitHub 저장소 → **Actions 탭 → "CRM 메일 자동 수집" → Run workflow** 로 즉시 한 번 실행.
초록색 체크가 뜨고, CRM 거래처 타임라인에 최근 메일이 나타나면 성공입니다.
이후에는 매시간 자동 실행됩니다.

## 자주 묻는 것
- **거래처 추가하면?** CRM에서 거래처 등록 시 "이메일 도메인"만 채우면 다음 시간부터 자동 수집됩니다.
- **본문 전체가 저장되나요?** 앞 500자만 저장됩니다 (요약 용도). 전문은 메일함에서 확인하세요.
- **Secrets 없이 돌아가면?** 아무것도 하지 않고 조용히 끝납니다 (오류 아님).
