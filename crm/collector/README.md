# CRM 메일 자동 수집기

GitHub Actions가 **1시간마다** 등록된 메일 계정(IMAP)에 접속해, CRM 거래처
도메인과 주고받은 메일을 CRM 타임라인에 자동 기록합니다.

## 메일 계정 등록 — CRM 설정 화면에서! (GitHub 불필요)

**https://mes.orocorp.kr/crm/ → 사이드바 ⚙ 설정 → "+ 메일 계정 추가"**

- 네이버 메일 / 네이버 웍스 버튼을 누르면 서버 주소·포트가 자동 입력됩니다.
- 다른 메일(다음, 회사 메일서버 등)도 IMAP 서버·포트를 직접 입력해 등록 가능.
- 아이디·비밀번호(2단계 인증 시 **앱 비밀번호**)를 넣고 저장하면 다음 시간부터 수집됩니다.
- 계정별로 수집 중지/재개·수정·삭제 가능.

⚠️ 비밀번호는 서버 DB(crm_mail_accounts)에 저장되며 CRM에 로그인한 팀원이 볼 수
있습니다. 실제 비밀번호 대신 **애플리케이션 비밀번호**를 발급해 쓰는 것을 권장합니다.

### 메일 쪽 사전 설정 (한 번만)
- **네이버 메일**: mail.naver.com → ⚙ 환경설정 → POP3/IMAP 설정 → "IMAP/SMTP 사용" 켜기.
  2단계 인증 사용 시 네이버 내정보 → 보안설정 → 애플리케이션 비밀번호 발급.
- **네이버 웍스**: 웍스 메일 → 환경설정 → IMAP/SMTP 사용 켜기 (관리자 허용 필요할 수 있음).

## 동작 방식
1. CRM 설정 화면에 등록된(사용 중) 메일 계정 목록을 읽음
2. 각 계정의 받은편지함 + 보낸편지함에서 최근 3일치 메일 확인
3. 보낸/받는 사람 도메인이 거래처의 "이메일 도메인"과 일치하면 타임라인에 기록
   - 거래처에서 온 메일 → "받음", 우리가 보낸 메일 → "보냄"
   - 메시지 ID 기반 중복 방지, 본문은 앞 500자만 저장
4. 거래처를 새로 등록할 때 "이메일 도메인"만 채우면 자동으로 수집 대상에 포함

## GitHub Secrets (저장소 관리자용)
| Secret | 필수 | 설명 |
|---|---|---|
| `SUPABASE_SERVICE_ROLE_KEY` | ✅ | Supabase 대시보드 → Settings → API → service_role |
| `NAVER_MAIL_USER/PASSWORD`, `WORKS_MAIL_USER/PASSWORD` | 선택 | 예전 방식(보조). 설정 화면에 같은 계정을 등록했으면 지워도 됨 |

## 실행 시점
- 매시 5분 자동 + 수집기 코드 변경 push 시 1회 + Actions 탭에서 수동(Run workflow)
